## 수행한 작업

- 저장소 안의 Markdown 파일 전체를 확인했다.
- 원고 파일, 부록 후보, 기존 관리문서, 중복/병합 후보를 분류했다.
- 최종 책 구조 기준으로 각 파일의 추정 장 배치를 `00_management/chapter_status.md`에 정리했다.
- 요청에 따라 원고 본문 파일은 수정하지 않았다.

## 핵심 판단

현재 원고는 최종 목차의 대부분을 이미 포함하고 있다. 다만 기존 폴더 구조는 `구조로 공부하다 / 사고를 구현하다 / 의료를 구조화하다` 중심이라, 최종 책 구조의 6부 체계와 장 번호가 맞지 않는다. 다음 단계에서는 파일 이동/복사보다 먼저 최종 목차 기준의 `book_manuscript` 구조를 따로 만드는 편이 안전하다.

실제 병합이 필요한 후보는 두 묶음이다.

| 후보 | 판단 |
|---|---|
| `AI를 도구로 쓰는 공부` / `AI를 공부에 쓴다는 것` | 하나의 10장으로 통합 권장 |
| `Streamlit, 그리고 Jisong Cloud` / `작업용 개인 클라우드, Jisong Cloud 6.0` | `Jisong Cloud 6.0`을 기준본으로 두고 구버전 내용을 일부 흡수 권장 |

기존 상태표에서 언급된 `CleanEMR 2`, `모델은 처음부터 맞지 않는다 2`, Obsidian PKM 추가 변형, `어떤 의사가 될 것인가` 별도 브런치 버전은 현재 저장소의 Markdown 파일 목록에서는 발견되지 않았다. 따라서 지금 단계에서는 삭제나 선택을 하지 않고, 현재 발견된 파일을 기준본으로 두는 것이 맞다.

## 확인이 필요한 부분

- 최종 30장 `판단은 설명될 때 완성된다`는 독립 Markdown 파일이 없다.
- 몇몇 긴 의료 파트 원고는 첫 줄이 `#`만 있거나 빈 소제목이 있어 책 원고화 단계에서 제목/헤딩 정리가 필요하다.
- `Jisong Cloud 6.0` 원고 첫 줄에 불필요한 문자처럼 보이는 `\"}`가 있어 다음 편집 단계에서 확인해야 한다.
- `VoiceGrape` 원고의 소제목 번호가 `1, 3, 3`으로 보인다.
- Brunch 메타데이터처럼 보이는 `title:` 첫 줄이 있는 문서가 몇 개 있다.
