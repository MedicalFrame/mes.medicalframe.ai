# Turn 4 편집 보고서 (최종 산출물 준비 및 합본)

## 1. 마크다운 자동 병합 및 `book.md` 생성
`build_book.py` 스크립트를 작성하여 흩어져 있던 `book_manuscript/` 하위의 모든 마크다운 파일들을 하나의 통합 파일인 **`book.md`**로 병합 완료했습니다.

병합 순서는 목차(`README.md`)에 정의된 순서와 완벽히 일치합니다:
- 프롤로그 -> Part 1 ~ Part 6 -> 에필로그 -> 부록(Appendix)

## 2. DOCX 변환을 위한 양식 및 마크다운 정제
DOCX 변환기(Pandoc 등)를 통과할 때 레이아웃이 깨지지 않도록 아래의 위험 요소들을 스크립트를 통해 자동으로 정제했습니다.
- **페이지 분리 (Page Break):** 각 챕터와 Part 간지가 시작될 때 DOCX 포맷에서 새 페이지로 인식할 수 있도록 `<div style="page-break-before:always;"></div>` HTML 태그를 삽입했습니다.
- **이미지와 캡션 간격:** `![이미지]` 태그 바로 위/아래에 텍스트가 붙어 있으면 렌더링 오류가 발생할 수 있어, 이미지 위아래로 반드시 빈 줄을 하나씩 추가하는 로직을 적용했습니다.
- **연속 빈 줄 최적화:** 의도적으로 과도하게 넣은 빈 줄(`\n\n\n` 이상)은 빈 페이지를 유발하므로 최대 2개(`\n\n`)로 축소 평탄화했습니다.

## 3. 최종 출력 시 저자 확인 사항 (Checklist)
`book.md`를 DOCX 또는 PDF로 내보내신 후, 다음 항목들을 마지막으로 꼭 점검해 주십시오.

- [ ] **페이지 브레이크 점검:** 챕터 H1 제목이 페이지 중간에서 시작하지 않고 새 페이지 상단에 잘 위치했는가? (HTML 태그가 정상 인식되었는지 확인)
- [ ] **이미지 해상도 및 가운데 정렬:** 모니터에서는 잘 보이던 이미지가 A5/B5 페이지 폭을 넘어 잘리거나 해상도가 깨지지 않는가? 
- [ ] **인용문 블록 스타일:** `>` 마크다운 인용구가 본문과 구별되는 적절한 폰트/들여쓰기 디자인으로 렌더링되었는가?
- [ ] **판권기 및 속표지 레이아웃:** 파일 최상단에 비워둔 '판권기 및 목차' 공간을 수동으로 편집하여 책의 구색을 맞추었는가?
