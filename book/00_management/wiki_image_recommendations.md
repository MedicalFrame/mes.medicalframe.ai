# 사진 추가 추천 위키

## 목적

- 원고의 논리 전환이 큰 지점에 시각적 쉼표를 넣는다.
- 설명형 장에서는 “문장으로만 설명되는 구조”를 도식이나 화면 캡처로 보강한다.
- 이미 충분히 스크린샷이 많은 장은 이미지를 더 넣기보다 배치를 정리하는 쪽이 낫다.

## 우선순위 높은 추가 후보

### 1. `book_manuscript/01_왜_자꾸_모델을_만드는가/04_의료를_구조화한다는_것.md`

- 추천 이미지:
  - `감각 → 수치 → 관계 → 모델 → 예측 → 설명 가능한 판단` 흐름도 1장
- 넣는 위치:
  - `### 4) 예측과 설명으로 연결하기` 바로 앞 또는 뒤
- 이유:
  - 이 장은 책 전체 선언문에 가까워서, 이후 장들을 관통하는 핵심 파이프라인을 한 장으로 보여주면 독자의 기억에 더 잘 남는다.

### 2. `book_manuscript/02_구조로_공부하다/02_암기와_이해는_다른_층위.md`

- 추천 이미지:
  - `이해할 것 / 암기할 것` 2분할 정리표
  - 또는 실제 노트 일부를 익명화한 예시 1장
- 넣는 위치:
  - `### 2) 암기할 것과 이해할 것을 나눈다` 아래
- 이유:
  - 이 장은 메시지는 명확하지만 전부 추상 문장이라, 구분 기준을 시각적으로 보여주면 설득력이 훨씬 커진다.

### 3. `book_manuscript/02_구조로_공부하다/03_실습은_사고의_전환점.md`

- 추천 이미지:
  - `암기 → 실습 → 재구성 → 더 깊은 이해` 순환 도식
- 넣는 위치:
  - `### 2) 반복은 이해를 다시 세운다` 아래
- 이유:
  - 현재는 전부 회고적 산문이라 호흡이 조금 길다. 작은 순환 다이어그램 하나만 있어도 장의 구조가 눈에 들어온다.

### 4. `book_manuscript/03_사고를_구현하다/01_사고를_구현한다는_것.md`

- 추천 이미지:
  - 프로젝트 맵 1장
  - 예: `HAOS / Jisong Cloud / OpenClaw / VoiceGrape / NeuroFrame / EstroFrame`
- 넣는 위치:
  - `### 2) 자동화는 사고의 외부화다` 아래
- 이유:
  - Part 3 전체의 안내문 역할을 하므로, “이 뒤에 어떤 프로젝트들이 나오는지”를 한눈에 보여주는 내비게이션 이미지가 효과적이다.

### 5. `book_manuscript/06_AI_시대에도_남는_것/01_판단은_설명될_때_완성된다.md`

- 추천 이미지:
  - 책 전체 결론을 요약하는 1장 도식
  - 예: `모델은 정답이 아니다 / 설명이 판단을 완성한다 / 책임은 사람에게 남는다`
- 넣는 위치:
  - 장 첫머리 또는 `### 3) 설명은 신뢰와 책임을 만든다` 아래
- 이유:
  - 결론 장은 짧고 밀도가 높아서, 압축된 도식 한 장이 들어가면 마지막 인상이 또렷해진다.

## 넣으면 좋은 보조 후보

### 6. `book_manuscript/06_AI_시대에도_남는_것/02_어떤_의사가_될_것인가.md`

- 현재도 이미지가 있지만, 성격상 “개념 그림”보다 “세 축 요약 도식”이 더 잘 맞는다.
- 이미지를 유지한다면:
  - `성장 / 환자 / 자기 자신(自信)` 3축 구조가 한눈에 보이도록 재구성된 한 장이 가장 좋다.

### 7. `book_manuscript/99_appendix/02_Obsidian_PKM_연결을_관리하기.md`

- 추천 이미지:
  - vault 구조 캡처 1장
  - index 문서와 폴더 경계를 보여주는 예시 1장
- 이유:
  - PKM을 안 쓰는 독자에게는 개념이 추상적일 수 있어, 실제 구조 예시가 이해를 돕는다.

## 추가보다 정리가 더 중요한 장

### `book_manuscript/03_사고를_구현하다/04_작업용_개인_클라우드_Jisong_Cloud_6_0.md`

- 이 장은 이미 이미지가 꽤 많다.
- 새 이미지를 추가하기보다:
  - 각 이미지의 역할을 분명히 하고
  - 이미지 직후 설명 문장을 짧게 정리하고
  - 중복되는 화면 캡처는 줄이는 편이 낫다.

### `book_manuscript/04_기록은_곧바로_데이터가_되지_않는다/03_로컬_기반_EMR_파싱_CleanText.md`

- 이미 스크린샷 밀도가 충분하다.
- 추가보다 우선할 일:
  - 화면 캡처의 순서를 `입력 → 처리 → 비식별화 → 결과` 흐름으로 더 선명하게 보이게 재배치
